# 修复计划：用户体验优化

## 业务背景

用户反馈了 5 个明显的产品体验问题，影响用户使用：

1. **广场页面点赞后整页刷新** - 动画播放到一半突然中断，用户体验差
2. **首页控制台报错** - 打开首页就有错误，可能有隐患
3. **卡片视觉不对称** - 布局不合理，视觉效果差
4. **发布页面刷新按钮不可见** - 黑色背景上黑色按钮，用户找不到
5. **Top3 页面日期显示 Invalid Date** - 数据展示错误

## 技术方案

### 1. 点赞刷新问题

**根因分析**：
- `WishCard.svelte` 第 80 行调用 `onLike?.()`
- 触发父组件 `+page.svelte` 的 `handleLike` 函数
- 该函数调用 `loadTop3()` 重新加载整个列表
- 导致组件重新渲染，动画中断

**解决方案**：
- 移除 `onLike?.()` 调用
- 在卡片内部直接更新本地状态
- 不触发父组件重新渲染

### 2. Invalid Date 问题

**根因分析**：
- `HourlyTopItem` 接口没有 `created_at` 字段
- `formatRelativeTime` 函数接收到 `undefined`
- `new Date(undefined)` 返回 Invalid Date

**解决方案**：
- 在 `HourlyTopItem` 添加可选字段 `created_at?: string`
- 更新 API 返回该字段
- 在函数中添加日期有效性检查

### 3. 卡片布局问题

**用户建议**：
> 可以把最底部的一条（心愿密钥和时间）放到小卡片上面，内容居中，下面放两个按钮

**解决方案**：
- 调整 DOM 结构：心愿文字 → 密钥时间 → 点赞分享
- 居中显示密钥和时间
- 添加上下边框形成视觉分隔

### 4. 刷新按钮不可见

**根因分析**：
- `.regenerate-btn` 的 `background: transparent`
- 在深色背景上几乎看不见

**解决方案**：
- 使用 `rgba(255, 255, 255, 0.15)` 白色半透明背景
- hover 时更亮

## 风险评估

| 风险项 | 影响 | 应对措施 |
|--------|------|----------|
| 类型变更 | 低 | 使用 Partial<> 兼容两种类型 |
| API 变更 | 低 | 新增可选字段，向后兼容 |
| 样式变更 | 低 | 仅调整布局，不影响功能 |
